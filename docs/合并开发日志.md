## 📌 Entry #8 — 2026-06-02 — ⭐ 给前端 Agent:修音换音色为什么是「三个输入」(UX 设计说明)

> 老板看 GUI 时问:「不应该只有一个原声调一个目标声调吗?第三个是什么意思?」——这是直觉误区。
> 本条把设计讲清楚,**将来重做前端的 Agent 必读**(决定这一页的交互)。决定:**保留三输入,不改**。

### 一句话
这页是**两步合成**(不是单纯 autotune):**第一步修音准**用 ①+②,**第二步换音色**用 ③。老板直觉的"原声+目标"只覆盖了第一步。

### 三个输入的角色(+ 对应后端函数)
| GUI 输入 | 作用 | 喂给 | 一句话 |
|---|---|---|---|
| **① 原始清唱(跑调)** | 要被修的对象(用户唱的) | `vocal.correct(source=①)` | 「**修谁**」 |
| **② 去和声参考** | **只取音高/旋律**(从原曲分出的干净主唱);**借音高,不借嗓音** | `vocal.correct(ref=②)` | 「**唱成什么调**」 |
| **③ 你自己的清唱** | 用户**自己音色**的样本,当 SVC 的"音色锚" | `vocal.voice(target=③)` | 「**听起来像谁**」 |

管线:`① + ② → correct()(修音准,24k)→ resample 44k → voice(source=修音输出, target=③)→ 成品`。代码见 `musicmaster/vocal/pipeline.py::two_stage`。

### 为什么不能只要两个(关键,别退回两输入)
只有 ①+② 时有两种坏结局,都是项目踩过的坑:
1. **只按 ② 修音高** → 还是用户那条略糙/偏弱的原声(纯 autotune 天花板;主项目实测全曲"一般般",见 `singer/调度开发日志.md` Entry #24)。
2. **拿 ② 当嗓音来源** → 成品**听起来像原唱那个人**,不像用户本人 —— 老板明确否掉:「没一个和我音色有关系,什么鬼」(见 Entry #29 → 产品方向锁定为**身份保留型美化**,SVC 目标音色 = 用户自己)。

∴ 第二步必须有 **③ 用户自己的干净嗓音**当锚,把声音**用用户音色重新生成**(顺带洗掉修音残留的电音/瑕疵、补强弱唱)。
**最终 = 在调(来自②)+ 干净(来自重生成)+ 还是你(来自③)。记忆法:② 决定唱成什么调,③ 决定听起来是谁。**

### 前端可做的简化(本次老板选择不做,留给将来)
- **③ 可选,默认 = ①**:若用户原始清唱本身够干净(只跑调、不吵),可同时当"要修的"和"音色锚",常规只需传 2 个文件。
- **加"纯修音"模式**:只用 ①+② 调 `vocal.correct()`(不走 SVC),输出"你本人原声但在调"。适合不想重生成音色的用户。
- 两者都不改变现有后端,只是 UI 层组合 `correct()` / `voice()` / `two_stage()` 三个现成函数。

### 血泪红线(前端别破坏)
